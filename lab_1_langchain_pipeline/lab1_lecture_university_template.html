The Pipe Operator

LCEL keeps the transformation readable: prompt -> model -> parser.

ChatPromptTemplate

Frames the input question or extraction request.

|

ChatOpenAI

Runs the model call with one shared configuration.

|

StrOutputParser

Returns plain text that later steps can reuse or store.

Why LCEL Here
  • Swap one component without rewriting the pipeline around it.
  • Batch the same runnable across professors with a single interface.
  • Keep chain boundaries visible when students read the notebook.
  • Use direct model calls only for multimodal OCR, not for every step.

Agents + MCP

Lab 1 finishes with a short payoff: the graph is already built, so the agent only needs to inspect and answer.

User Question Natural-language query from the notebook
Model Step Choose whether a tool call is needed
MCP Tool Call Schema check or Cypher execution
Tool Result Structured facts from Neo4j
Final Answer Response grounded in database state
Thought -> Action -> Observation stays visible because the notebook prints the ordered tool trace.
MCP standardizes tool exposure. The agent discovers Neo4j tools at runtime instead of relying on custom wrappers.
MultiServerMCPClient keeps the setup small. One connection object gives the notebook a reusable agent interface.
The loop stops early. The agent only keeps calling tools until it has enough evidence to answer.

13 Notebook Sections

The lab is intentionally linear: each section narrows the teaching goal and hands off to the next.

Setup Steps 1–4: environment, model config, imports, tiny LCEL example.
Crawl Steps 5–6: professor listing, default working set, one worked example.
Extract Steps 7–8: page OCR, header recovery, graph-ready merge preview.
Graph Steps 9–10: parallel professor pipelines, Neo4j load, direct verification.
Agent Steps 11–12: MCP tools, one answer loop, optional inline Gradio UI.

Carry These Into Every Lab

Lab 1 is narrow on purpose. The durable lesson is how to structure an AI workflow so it stays inspectable and reusable.

1
Deterministic vs. stochastic

Crawling and storage stay in plain Python; reasoning stays where models actually help.

2
Save intermediate artifacts

Keep notes, markdown, graph JSON, and traces so failure points stay visible.

3
One job per section

Students can understand the lab because each section has a narrow responsibility.

4
Reuse the same logic

Notebook, CLI, and optional UI all depend on the same underlying agent module.

5
Progressive disclosure

Lab 1 teaches the narrow path first; later labs add LangGraph and multi-agent patterns.
